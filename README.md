# Руководство по развертыванию проекта

## Требования
Для работы проекта требуется Debian-подобная ОС (Ubuntu/Debian/Mint).

## Установка
```bash
# Переходим в рабочую директорию
cd docker_vkm

# Собираем контейнеры
sudo docker compose build

# Запускаем сервисы
sudo docker compose up -d

# Логи конкретного сервиса
sudo docker compose logs -f [service_name]

# Все логи
sudo docker compose logs -f


/logs/                   # Логи nginx
/backend-api/            # backend-api
/backend-api/backups/    # Бекапы
/frontend-api/           # backend-api

## Проверка корректности базы данных

В корневой директории архива необходимо выполнить скрипт converter.py над базой данных сервиса переведенную в json при помощи manage.py loaddata. В результате получаем все необходимые таблицы в директории output_csv.
Далее необходимо удалить из предоставленной базы данных идентификатор под названием "Счётчик". А затем нужно отформатировать даты под формат хранения в json с помощью скрипта formmating_dates.py. 
Далее проверяем таблицу штаммов на схожесть с предоставленной нам таблицей штаммов с помощью скрипта compare.py.
